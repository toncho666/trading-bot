1. Пользователь пишет промт на создание стратегии и ее первоначальной валидации/тестирования - проверки на примемлемость. Промерка на примемлемость - доходность выше 1%

2. Сгенерированный файл кода стратегии помещается в папку strategies

3. Стратегия запускается раз в час при помощи .github/workflow. При этом данные для расчета берутся за предыдущий полный час

4. В случае появления сигнала (buy/sell) оптравляется запись в БД и сообщение в tg группу
